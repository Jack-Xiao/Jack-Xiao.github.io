git 的使用

发表于 2018-04-14   |     |   阅读次数 
当你被面临的问题困住的时候, 你就能感同身受的体会到那些技术的巨大优越性了, git 就是这样的技术

区

git有两个区, 暂存区, 工作区。

  • 工作区就是直接写代码,修修改改的地方。

  • 暂存区就是add . 命令所存放文件的地方。也是

    git stash --把工作区放到暂存区
git stash apply --把暂存区内容返回到工作区
git stash pop --把暂存区内容返回到工作区,并删除原有暂存区内容。

分支

  1. 创建分支

    git branch <branch>

  1. 切换分支

    git checkout <branch>

  2. 合并分支

    git merge <branch>

  3. 删除分支

    git branch -d <branch>

远程仓储

git push origin dev -- 推送对应的远程分支

$ git checkout -b dev origin/dev  (把远程存在的分支，clone 到本地)

git remote -v 查看远程库信息

git push origin dev --把本地仓储推送到远程仓储

2016年度个人工作总结报告

发表于 2017-01-02   |     |   阅读次数

时间飞逝，2016年就这样过去了。

这半年多的时间里，我的工作主要是维护易起行android项目,保持项目的活力。

从开始使用GA，进行android、iOS端的数据统计。到后来与营销部的同事讨论起数据的准确性，并不断优化。

从开始的不知cordova为何物，到后来慢慢熟悉，并阅读框架源码。在前端的插件在框架下无法使用时,通过修改源码,使其能够兼容我们的cordova框架,让插件也能够正常使用。

在我们项目随着业务不断扩大,原有旧版本框架已很难继续维护时,能及时的升级我们的cordova框架到最新版本,让版本平缓的过度升级。

在后来丰富我们的项目,使其增加更多的native特性,满足业务需求。

Corodva作为一个跨平台的解决方案，它是我们项目的起始点。它能使版本的迭代更加快速,与backbone框架的结合也使项目的mvc结构更清晰,项目的侵入性也更小。但作为几年前的跨平台解决方案，它存在不少问题，如代码量的巨大,速度慢,不好的使用体验(特别在低端机型)。

关注最近的移动端开发就发现,混合开发将成为常态。其开发周期与版本迭代能力不言而喻。而其中的佼佼者就是react-native,其次是阿里的weex(weex还很不成熟)。

在领导的安排下学习RN, 并尝试开发Demo。其近乎原生的体验,以及相对cordova少不少的代码,体验感觉更好。RN非常适合中小型项目。（但仍有不少坑(目前版本为0.39)）。

很荣幸能够进入广之旅, 并与大家一同共事、成长。

工作就像场马拉松,2016的结束只是路途中的一个拐点。

2017共勉。
