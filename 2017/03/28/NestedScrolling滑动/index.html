NestedScrollView 嵌套ViewPager 在嵌套 RecyclerView

根据业务的需要, 设想了一种NestedScrollView中 嵌套ViewPager, 在ViewPager中继续嵌套RecyclerView，然而当RecyclerView包含多个ItemType时,左右滑动ViewPager时就会产生问题。

于是换成一种NestedScrollView 包含多个RecyclerView, 实现不同的布局

关于截断

android:focusable="true"  
android:focusableInTouchMode="true"

该属性设置可以截断子级控件的默认行为,比如子控件EditText自动获取焦点,关闭键盘, 关闭键盘也可以使用下面这方法:

EditText对象的clearFocus();
InputMethodManager imm = (InputMethodManager)getSystemService(Context.INPUT_METHOD_SERVICE); 
imm.hideSoftInputFromWindow(editMsgView.getWindowToken(), 0);（关闭软键盘。。。）

NestedScrollView嵌套RecyclerView时,RecyclerView 自动指向自己的第一项,此时我们就可以使用

android:focusable="true"  
android:focusableInTouchMode="true"

截断该行为,从NestedScrollView最开始的地方显示。

关于NestedScrollView 嵌套RecyclerView

对于RecyclerView 的adapter中的Item高度, 要设置wrap_content。

滑动问题

1)recyclerview在support library 23.2.0 (or above)以上
2)recyclerview高度设置wrap_content
3)recyclerView.setNestedScrollingEnabled(false)

onMeasure 中是绘制view的动作, 一般在其中可对于item进行计算.
