记一次给google/smaple提交issue

发表于 2016-05-17   |     |   阅读次数

前言：无意中给google sample 项目提交了一个issue.

发现问题

前几天，看到有人提到google sample MVP的使用，于是进入github，下载源码查看。然而当我download下来运行时出现了报错，google / stackoverflow 都搜索不到相关的问题。

提交issue

解决不了问题，这很郁闷。于是我重新上github 查看项目，翻翻issue。看是否有人和我遇到一样问题，但找了几遍都没有。于是突然想到为什么不可以自己提一个issue呢，因为我是真的遇到了问题，并且用了各种办法也没有解决。于是我就提交了一个issue

问题初步解决

第二天有人回复遇到了同样的问题，然而一小时之后，他提到了一个解决办法。这里为何我没有想到这办法呢。自省为我对android studio理解的还不够透彻。一直都知道proguard-android.txt 与android SDK的混淆规则有关，然而我却完全忽略这一点，没有深入考虑。当我很高兴的说终于解决了问题了。然而google sample code 的ower JoseAlcerreca却强调

Again, disabling the minifier doesn't solve the problem.

When you say base model do you mean todo-mvp?

Can you try with 2.1.0?

他是对的。之后JoseAlcerreca 似乎也不知怎么办，反而问我是否也是用windows OS。我知道这时他也不知道这问题怎么解决了。之后过了几分钟 Jose提到

I've reported the problem to the build tools team.

聪明之人应该珍惜自己的时间，并且相信其他人。保证自己的高效性，并谦虚的向他人请教。

遇见build tools team

第一位benol 提到

Hi, this is Michal from the build tools team. I can't reproduce the
problem, I tried git commit 10fbc69 in android-architecture, todo-mvp 
branch - and it built just fine. Are you using the same branch/project/
commit? If not, can you post the commit number where you see the problem?

第二位 malmstein 说

These two lines in the default proguard-android.txt file are the issue:

void set*(%[]);
%[] get*();;

As a quick fix this is how I made it work in the Content Providers sample:

https://github.com/googlesamples/android-architecture/blob/dev-todo-mvp-contentproviders/todoapp/app/proguard-android.txt

Basically, grabbing the default proguard-android file and adding it to the folder:
https://github.com/googlesamples/android-architecture/blob/dev-todo-mvp-contentproviders/todoapp/app/build.gradle#L22

第一位说自己没办法重现，并想确认我的版本(我知道这是没有办法的回复)。 从第二位的回复来看，是他重现了这个问题，并找到了解决的办法。我照着他提供的文件，更新了proguard-android。问题得到了解决。

问题解决

最后回来，the builds tools team 的benol

I can reproduce now, thanks. It's a known issue with the parser in 2.1,
it's been fixed for 2.2.

负责google sample 的Jose 则说：

Please replace

minifyEnabled true

with

minifyEnabled false

for the debug buildType, until 2.2 is out.

最终承认build tools 2.1 有bug， 会在2.2 修复它。

红色的bug 标识，代表着我曾经的一段付出（难怪都喜欢开源，这种付出的确使人得到了很大的满足）。

写在最后的话：学海无涯，学无止境。
