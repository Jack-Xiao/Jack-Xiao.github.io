1.个推推送

最近在做推送通知，经历各种推送坑后，最终使用个推推送。然而个推标签的发送策略并没有交集、并集的概念。以下是个人对个推推送的标签功能的补全，实现交集并集功能的发送。

  • 要实现的功能是支持多个 ‘或’ 和 ‘并’ 功能。如符号 ‘||’ ‘&&’ 的功能。
  • 考虑到以后的扩展，支持多个标签的&&功能。如标签d1、d2、u0、a5、a7, 服务器发送的数据可能就为d1||d2&&u0||&&a5||a7 .可使用字符串的split。<!—more—>-

具体代码如下：

//取本地的标签列表
nativeTags = DefaultShared.getString(Config.GT_PUSH_TAGS, Config.DEFAULT_PUSH_TAGS);
//保存在列表中
List<String> nativeList = Arrays.asList(nativeTags.split(","));
//从推送中取到服务器发送过来的标签字符串
serverTags = valueMap.get(TAG_LIST);
//取得多个并集的数组
String [] serList = serverTags.split("&&");
//以下为关键代码，
for(int i =0;i < serList.length;i++){
   String [] currentArrayTag = serList[i].split("\\|\\|");
                        curContains = false;
                        for(int j=0;j<currentArrayTag.length;j++){
                             //判断本地标签列表是否有满足当前并集序列中的元素
                            if(nativeList.contains(currentArrayTag[j])){
                                curContains = true;
                                break;
                            }
                        }
                        if(!curContains) return;
}

代码完成，这样就可以实现个推推送中没有提供的交集与并集的功能。

##2.发送通知
发送通知时，要注意PendingIntent 参数：

PendingItent.getActivity(App.getContent,0,intent,PendingIntent.FLAG_UPDATE_CURRENT)

第四个参数为关键，PendingIntent.FLAG_UPDATE_CURRENT 否则收到的带有参数的通知就为旧的数据。

3.微信的签名

android中使用java 工具可获取签名文件的sha1 和 md5数据。一般的要求中可能只要用sha1的相关信息即可。但在微信分享的签名中需要提供的却是md5数据，并且去掉冒号(:) 并且全部变成小写的字符串。
于是我们可以先使用命令 mac下

cd ~/.android
keytool -list -v -keystore ~/App/keystore/yourkeystore.keystore

可以看到一些相应的信息 sha1 及md5 的数据。然后我们可以把md5 的数据copy下来，使用最近学习的python脚本把这些冒号去掉，并且把字符全部变成小写

source = "CE:E8:B7:1C:55:20:0C:A1:37:92:AF:1F:37:37:B3:28"
source = source.replace(":","")
source = source.lower();
print source
